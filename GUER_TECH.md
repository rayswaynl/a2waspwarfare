# IMPLEMENTATION REFERENCE (b75 — GUER kill-based tech + FOB system)

Built on branch `GUER_Upgrade_System`, one commit per feature (b75.1 … b75.11) + two review-fix commits
(b75.12 FOB correctness, b75.13 polish). Only the `[55-2hc]…chernarus` mission is hand-edited; the Takistan
mirror is regenerated by `Tools/LoadoutManager` (`dotnet run`) on every run (the `7za environment variable is
not set` error AFTER "TAKISTAN DONE!" is EXPECTED — ignore it).

## Core mechanic
`WFBE_GUER_PLAYER_KILLS` = cumulative kills made by resistance PLAYERS vs WEST/EAST. Server-incremented in
`Server/PVFunctions/RequestOnUnitKilled.sqf` (one per qualifying kill; victim gated to west/east),
`publicVariable`'d, isNil-seeded 0 in `Common/Init/Init_CommonConstants.sqf`, JIP-pushed in
`Server/Functions/Server_OnPlayerConnected.sqf`, and re-broadcast each `Server/Server_GuerStipend.sqf` cycle.
This REPLACES the old elapsed-time tier (b75.5 removed it).

## Kill values (tunable constants in Init_CommonConstants.sqf — "you come up with the kill values")
- Vehicle tiers: `WFBE_C_GUER_KILLTIER_1=15` (BRDM-2 + T-34), `_2=40` (T-55), `_3=80` (T-72 + BMP-2; on Takistan
  tier-3 is a ZU-23 Ural — GUE has no T-72/BMP-2 there).
- M113 VBIED: `WFBE_C_GUER_VBIED_M113_KILLS=25`; type `M113_UN_EP1`; `_SPEEDCOEF=2.0` (driver-local boost loop,
  forward-axis only).
- Ka-137 flares: by `WFBE_GUER_VEHICLE_TIER` → `WFBE_C_GUER_KA137_FLARE_MAGS` = 60 / 120 / 240 Rnd_CMFlareMagazine.
  NB: A2-OA has NO 30Rnd flare mag, so the floor is 60Rnd (closest valid class) — the count still rises by kills.
- Barracks AI cap: `WFBE_C_GUER_BARRACKS_AI_BASE=4` + 1 per `_PER_KILLS=10` kills, capped `_MAX=12` (engine group
  limit). Applied in GUI_Menu_BuyUnits.sqf and mirrored on the RHUD + Upgrade Center.

## FOB system
- `WFBE_GUER_FOB_AVAIL=[barracks,light,heavy]` tokens, +1 per destroyed enemy factory of that type at BOTH choke
  points: `Server/Functions/Server_BuildingKilled.sqf` (Path A, normal kills) and
  `Server/Functions/Server_HandleSpecial.sqf` (Path B, VBIED — the null-instigator blast is excluded from Path A,
  so no double count). missionNamespace var, publicVariable'd + re-broadcast (same durability as the kill counter).
- FOB trucks (`WFBE_C_GUER_FOB_TRUCKS`, worldName-branched: Ural_INS/UralOpen_INS/GAZ_Vodnik on CH,
  Ural_TK_CIV_EP1/V3S_Open_TK_CIV_EP1/V3S_TK_EP1 on TK). They are NOT registered in Core_GUE (those classnames are
  shared with active factions and Core_GUE would clobber their tuples — b75.12). They ride their CANONICAL
  registration (the GUER depot has no faction filter + bypasses the upgrade gate, so they still show), are
  relabeled "FOB (…)" in `Client/Functions/Client_UIFillListBuyUnits.sqf` (resistance-only), and appended to the
  depot pool in `Common/Config/Core_Root/Root_GUE_PlayerOverlay.sqf` only while the matching token > 0. A GUER FOB
  instance is distinguished from an AI faction's same-class truck purely by the broadcast `wfbe_is_guer_fob` flag.
- "Build FOB" action: `Common/Init/Init_Unit.sqf` (on every client) → `Client/Action/Action_BuildFOB.sqf` →
  `Server/PVFunctions/RequestFOBStructure.sqf` (registered in `Common/Init/Init_PublicVariables.sqf`). It
  re-validates the token + the enemy-town/base/water gate (`WFBE_FNC_GuerFobBlocked`, both enemy sides), spends one
  token, runs the STANDARD GUER construction (side resistance → factory registers in wfbe_structures → forward
  spawn + production), and consumes the truck. Unlimited FOBs; no base-area cap.
- FOB trucks AND built FOB factories are mobile/forward spawn points "like a town"
  (`Client/Functions/Client_GetRespawnAvailable.sqf` + `Client_OnRespawnHandler.sqf`).

## UI / feedback
- RHUD (`Client/Client_UpdateRHUD.sqf`): GUER-only "Tech Kills" row directly ABOVE the "FOB: B n | LF n | HF n" row
  (reuses spare control pairs 15/16 + 17/18, IDC 1360-1363).
- Unlock notifications: server sets `WFBE_GUER_UNLOCK_MSG` on each threshold crossing; the watcher in
  Root_GUE_PlayerOverlay.sqf shows it once (no stale re-pop on JIP).
- Upgrade Center (`Client/GUI/GUI_UpgradeMenu.sqf`): enabled for GUER in `Client/GUI/GUI_Menu.sqf`, repurposed as a
  READ-ONLY kill-tech board (kills, tiers + next thresholds, M113, flares, barracks AI, FOB availability).
- Distinct GUER-FOB kill message: `GuerFobCleared` case in `Client/PVFunctions/LocalizeMessage.sqf` (same bounty,
  different wording); routed from Server_BuildingKilled.sqf, which also now cleans up destroyed resistance factories.

## Build times
GUER factory build times normalized to 60s (was 70/90/110/130/150/70/280) to match WEST/EAST, in all three GUER
structure configs (`Structures_CO_GUE`, `Structures_GUE`, `Structures_OA_TKGUE`). HQ stays 30.
