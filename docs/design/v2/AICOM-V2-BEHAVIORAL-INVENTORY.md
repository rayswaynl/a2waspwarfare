## Supervisor Modes And Transitions

The live supervisor is no longer just FULL/ASSIST/STOPPED. It has pre-active watchdog behavior, an AI build-grace warm-up, delegated maneuver under a human commander, and a locked-full override. The table names the behavior modes the V2 design has to preserve or intentionally replace.

| Mode | Transition trigger | Gate/state variables | Behavioral output |
| --- | --- | --- | --- |
| Pre-active watchdog | Before `_active` is evaluated. | `WFBE_C_AICOM_WATCHDOG>0`; supervisor heartbeat stale check has scan/cooldown gates. | May restart stale supervisor owner generation. No worker dispatch beyond the watchdog logic. |
| Stopped | `WFBE_C_AI_COMMANDER_ENABLED<=0` or side HQ missing/dead. | Active gate fails. | Sets `wfbe_aicom_running=false` once and suppresses all worker dispatch until the side is active again. |
| Full warm-up | AI is commander (`_humanCmd=false`) but build grace has not elapsed. | No human commander, or `WFBE_C_AI_COMMANDER_LOCK>0` forces AI ownership; `_canBuild=false` until `WFBE_C_AI_COMMANDER_BUILD_GRACE` passes. | Runs active-gated workers and maneuver if delegated/full, but economy/build workers remain off until grace expires. |
| Full economy | AI is commander and build grace elapsed. | `_humanCmd=false`, `_canBuild=true`, HQ alive, commander enabled. | Sets `wfbe_aicom_running=true`; dispatches all core economy/build/maneuver workers plus active side controllers. |
| Assist delegated | Human commander seated, war-room delegation on. | `_humanCmd=true`, `_aiDelegate=true`, so `_aiStrategy=true` while `_canBuild=false`. | Sets `wfbe_aicom_running=false`; Strategy/Snapshot/Allocate continue, AssignTowns is called, hybrid Teams/Produce may run, base/upgrade/MHQ/base-sell/beacon stay off. |
| Assist direct | Human commander seated, war-room delegation off. | `_humanCmd=true`, `_aiDelegate=false`, `_aiStrategy=false`, `_canBuild=false`. | Executor, intent, stipend, town call/self-gates, HC merge/top-up, hybrid refill, and disband can run; autonomous Strategy/Snapshot/Allocate and build/research stay off. |
| Locked full command | Human may be seated, but lock keeps AI in charge. | `WFBE_C_AI_COMMANDER_LOCK>0` forces `_humanCmd=false`. | Behaves as full warm-up/full economy depending on build grace; protects unattended eval/night soak behavior. |

Key side effects: `wfbe_aicom_running` is true only when the AI is treated as full commander; `_aiStrategy` is wider than `_canBuild`; build/research/base work remains under `_canBuild`; assist-mode body maintenance is handled by the hybrid Teams/Produce path when enabled.

## V2 Design Implications

- The replacement brain must model command ownership separately from build authority: `_aiStrategy` can be true when `_canBuild` is false, and assist mode can still maintain bodies.
- The eight-worker table is insufficient by itself; V2 must also account for watchdog, intent publishing, stipend, HC top-up/merge, hybrid refill, disbanding, MHQ relocation, optional beacons, and the Snapshot/Allocate passes.
- Wildcard truth must come from the live weight assignment and `_weights` table, not from the file header. Forced-zero cards are behaviorally important because their apply blocks still exist but cannot be selected.
- Several constants have early compatibility definitions and later guarded or map-specific definitions. V2 config migration should preserve override semantics instead of flattening by name without reading assignment kind and order.
- Zargabad-specific overrides in the constants file are part of the current behavior surface even though this inventory uses the Chernarus mission path as the canonical source root.
