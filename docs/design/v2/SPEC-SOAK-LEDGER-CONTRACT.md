# SPEC-SOAK-LEDGER-CONTRACT

Status: SPEC-READY. Implementation files are intentionally not created in this prep lane.

Guide rev for downstream PR bodies: GR-2026-07-03a.

## Objective

Define the append-only JSON Lines ledger consumed by the nightly soak farm, Discord verdict poster, and future admin hub.

Location:

```text
Tools/Soak/soak-ledger.jsonl
```

The first line may be a header comment:

```text
# a2wasp-soak-ledger-v1 jsonl; skip lines beginning with '#'
```

Every non-comment line is one complete JSON object.

## Row Identity

`rowId` is generated by the append helper, not by callers:

```text
YYYYMMDD-NNNN
```

`NNNN` is one-based and scoped to the file day. The helper finds the last non-comment row for the same UTC date and increments.

`stampId` is unique across successful rows. Duplicate `stampId` appends are rejected unless `status` starts with `SKIP_` and `allowDuplicateSkip=true`.

## Status Enum

Allowed `status`:

- `POSTED`
- `POSTED_LEDGER_ONLY`
- `DRY_RUN`
- `SKIP_NO_STAMP`
- `SKIP_BAD_STAMP`
- `SKIP_DUPLICATE`
- `SKIP_VERSION_MISMATCH`
- `SKIP_TOO_SHORT`
- `SKIP_BOX_DOWN`
- `FAIL_ANALYZER`
- `FAIL_LENS`
- `FAIL_LEDGER`

## Append Helper Contract

Future file:

```text
Tools\Soak\Append-LedgerRow.ps1
```

Parameters:

| Parameter | Required | Meaning |
| --- | --- | --- |
| `-LedgerPath` | Yes | JSONL file path. Create with header if absent. |
| `-StampPath` | Yes for normal rows | Deploy stamp JSON. |
| `-AnalyzeJsonPath` | No | Analyzer JSON path. |
| `-LensJsonPath` | No | Lens JSON path. |
| `-Status` | Yes | One status enum value. |
| `-ServerRptPath` | No | Copied server RPT path. |
| `-HcRptPath` | No | Copied HC RPT path. |
| `-DiscordGuildId` | No | Discord guild snowflake. Store as string. |
| `-DiscordChannelId` | No | Discord channel snowflake. Store as string. |
| `-DiscordMessageId` | No | Discord message snowflake. Store as string. |
| `-Note` | No | Repeatable note string. |
| `-AllowDuplicateSkip` | No | Allows duplicate `stampId` only for `SKIP_*`. |

Pester tests:

- Append two PASS rows and verify monotonic `rowId`.
- Append one skip row and verify KPI values are null, not zero.
- Reject duplicate successful `stampId`.
- Preserve and skip `#` comment lines.
- Validate `schema` field on every row.

## Admin Hub Access Contract

The future admin hub reads the ledger like this:

1. Ignore blank lines.
2. Ignore lines whose first non-space char is `#`.
3. Parse each remaining line as JSON.
4. Sort by `rowId` descending by default.
5. Treat `null` as unknown/not applicable and `0` as an actual zero.
6. Never infer PASS from missing fields.

## Schema Evolution

No bump:

- Add a nullable field.
- Add a new enum value only if old readers treat unknown values as `WATCH`.

Requires `a2wasp-soak-ledger-row-v2` plus migration script:

- Rename a field.
- Change numeric units.
- Change `rowId` format.
- Remove or repurpose a field.
