# WASP B59 → B60 Improvement Plan

## Headline
B59 is a deliberate 15-team load test and it is doing its job: server FPS softens to 35-38 at peak and antistack scales ~307→489ms with `allUnits`, but the round does not *progress* — both sides freeze in a DEFEND/DEFEND-looking stall while captures stop. The single biggest lever is the **memoryless spearhead picker** (`AI_Commander_Strategy.sqf:107-155`): it re-scores deterministically every ~60s with zero progress memory, so the same town is re-selected forever (live: EAST NWAF 460 / WEST Nizhnoye 1122 flat all round). The structural cause *underneath* that — why a fist arrives but never captures — is the **HC-team no-refill bleed** (`AI_Commander_Produce.sqf:63` skips 100% of live teams), which is a correctness bug, not a B60 toggle. The one risk to watch: do NOT let any of the strategy-layer or MHQ work introduce a standing-still or front-teleported AI — that violates the hard player-experience mandate, and the MHQ relocation is the single item that can lose a round outright.

## B60 go/no-go

**Start conservative — ship BOTH features default-OFF for the first soak, not default-ON.** `AI_Commander_Base.sqf:33` deploys the HQ once and never re-mobilizes, and the live RPT shows `AIR-INSERT=0 / PARA=0` — the flyInHeight host path the cannon-nudge rides has *never executed in the RPT lifetime*. Default-ON would make the first-ever execution of net-new control paths the soak itself, on a build already at 35-38 FPS peak. Use absent-defaults-false (`missionNamespace getVariable ["WFBE_C_...",0]`) so a missing param can never silently arm a feature.

- **MHQ relocation:** default-OFF behind `WFBE_C_AICOM_MHQ_RELOCATE`. When trialed: scope to **pre-factory-only** (`alive Light/Heavy structures == 0`) so it never orphans the factory cluster (`_findBuildPos` keys off `_hqPos`, `AI_Commander_Base.sqf:132-136`); take **all three** flags (`wfbe_hqinuse` waitUntil-then-set + `wfbe_hq_repairing`==false + HQ-alive recheck) at the moment of setPos; cap hop **≤1500m** (short of the 1752m Berezino legs that froze EAST); deadline/stuck fallback teleports **BACKWARD to last-safe node or re-deploys in place — NEVER forward**; abort if enemy within ~600m of destination, re-checked at deploy. Log `MHQ_RELOCATE` start/arrive/abort + HQ-near-enemy incident count. Accept that in its safe form it will frequently abort having moved nothing — that's correct; measure value before calling it a win. Decide with Ray whether the real goal is a mid-war forward FOB (a redesign) before committing past pre-factory scope.

- **Heli cannon-nudge:** **do not ship — reject the feature as currently premised.** It rides flyInHeight (transport-insert Spawn, never fired), but the *attack* Mi24_P is on a different path and is *already scoring kills unaided* (7 Mi24 KILL lines, EAST→GUER); the `madepilotscontrolunguidedrockets` addon (loaded, 2 hits) already manages muzzle selection; air has zero telemetry. There is no demonstrated problem to solve, and forcing low-altitude + doFire on the side's one slow-to-replace gunship trades a working kill-scorer for a likely shoot-down. **Do ship the decoupled half now:** the behaviour-neutral air telemetry line (`AICOMSTAT|...|CANNON_NUDGE|team|alt|tgtDist|fired`, or simply an attack-heli engagement/shoot-down line) so the air lane stops being invisible. Only revisit the nudge if that telemetry shows the gunship actually underperforming.
