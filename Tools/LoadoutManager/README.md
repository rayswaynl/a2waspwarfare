# LoadoutManager
<!-- GUIDE-REV: GR-2026-07-03a -->

Generates and mirrors SQF loadout and balance files from `Data/` into all three terrain
missions (Chernarus source → Takistan + Zargabad mirrors).

## Prerequisites

- .NET 8.0 SDK
- 7-Zip (optional; required only for `_MISSIONS.7z` server-deploy packing)

7-Zip is auto-detected in this order:
1. `7za` environment variable (path to executable)
2. `C:\Program Files\7-Zip\7za.exe` or `7z.exe`
3. `C:\Program Files (x86)\7-Zip\7za.exe`
4. `7za` / `7z` on PATH

If none is found, LoadoutManager logs a line and skips the pack step. The mirror still completes.
Set `A2WASP_SKIP_ZIP=1` to suppress packing explicitly.

## Usage

```powershell
cd Tools\LoadoutManager
dotnet run -c RELEASE                      # standard: mirror CH->TK+ZG, pack if 7-Zip present
dotnet run -c RELEASE -- --check           # dry-run: report TK drift without writing
```

Use `RELEASE` for all PR work. `SERVER_DEBUG` and `DEBUG` are development/diagnostic builds
only; never ship a mission built with `DEBUG`.

## Build configurations

| Configuration | Use |
|---|---|
| `RELEASE` | PR work and server deployment |
| `SERVER_DEBUG` | Diagnosing a live-server issue (logging on) |
| `DEBUG` | Local dev only — adds funds/tiers unlock; never deploy |
| `AIRWAR_RELEASE` | Air War module production build |
| `AIRWAR_DEBUG` / `AIRWAR_SERVER_DEBUG` | Air War development/diagnostic |

## What is mirrored

LoadoutManager writes `Common/Init/Init_CommonBalanceInit.sqf` and
`Client/Module/EASA/EASA_Init.sqf` (and related generated files) into all three terrain
folders. It does NOT mirror `mission.sqm`; lobby-slot changes require manual edits to all
three `mission.sqm` files.

## version.sqf.template restore

LoadoutManager may touch `version.sqf.template` on TK and ZG as a side effect. Always restore
both templates to the branch HEAD before staging:

```powershell
git checkout origin/claude/build84-cmdcon36 -- `
  "Missions_Vanilla/[61-2hc]warfarev2_073v48co.takistan/version.sqf.template" `
  "Missions_Vanilla/[61-2hc]warfarev2_073v48co.zargabad/version.sqf.template"
```

Verify the restored values — do not propagate CH values to TK or ZG:

| Field | CH | TK | ZG |
|---|---|---|---|
| `WF_MAXPLAYERS` | 34 (naval) | 61 | 61 |
| `STARTING_DISTANCE` | 7500 | 7500 | 5000 |
| `IS_CHERNARUS_MAP_DEPENDENT` | defined | NOT defined | NOT defined |
| `IS_NAVAL_MAP` | defined | NOT defined | NOT defined |

## Zargabad notes

ZG is a small map (~8 towns, 8192m). The `version.sqf.template` `STARTING_DISTANCE` is 5000
(not 7500). ZG-specific AICOM tuning constants live in the `if (worldName == "Zargabad")`
block at the top of `Common/Init/Init_CommonConstants.sqf` and are not generated by
LoadoutManager; they are hand-maintained in the ZG mirror.

## Error cases

- **`dotnet run` fails**: stop and tell the user to install the .NET SDK.
- **7-Zip not found**: LoadoutManager logs one line and skips `_MISSIONS.7z`. The mirror
  is complete. Install 7-Zip or set `A2WASP_SKIP_ZIP=1` to suppress the log.
